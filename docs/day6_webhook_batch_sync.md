# Day 6 Webhook, Batch Import, and Feishu Sync

## Goal

Close the research gap around real business entry and downstream sync:

- inbound email should not rely only on manual paste
- n8n should have a concrete webhook target
- Feishu should be represented as a downstream sync target
- timeline should record webhook, batch import, review, and sync events

## Changes completed on 2026-06-18

- Added `POST /api/webhooks/email` for n8n or email automation tools.
- Added `POST /api/leads/import-batch` with JSON payload generated by front-end file reading.
- Added `source_channel`, `sync_target`, `sync_status`, and `synced_at` to `leads`.
- Added `lead_events` table for timeline events.
- Added `POST /api/leads/{id}/sync/feishu` as the Feishu Bitable sync endpoint.
- Added batch file selector and `批量导入` button.
- Added `同步到飞书` button.
- Updated lead cards with source and sync status.
- Updated README with n8n webhook, Feishu API, and lead scoring explanation.
